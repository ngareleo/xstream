# TraceHistoryTab

Playback session history section in Settings. Displays a table of past playback
sessions with video title, resolution, start time, and trace ID. Each row
includes a "Copy" button to copy the trace ID to the clipboard for inspection
in Seq.

**Source:** `client/src/components/trace-history-tab/`
**Used by:** SettingsPage right-pane, conditional render when `section=trace`.
Wrapped in a page-level `<Suspense>` boundary.

## Role

Settings tab for observability. Surfaces playback history and links to
distributed traces. End users copy trace IDs and paste them into Seq to inspect
the full streaming pipeline (transcoding, buffering, errors) for a session.

## Queries & mutations

### Query: `TraceHistoryTabQuery`

```graphql
query TraceHistoryTabQuery {
  playbackHistory {
    id
    traceId
    videoTitle
    resolution
    startedAt
  }
}
```

- **Variables**: None.
- **Fetch policy**: `store-and-network` (render cached data if present, refetch
  in background).
- **Response**:
  - `playbackHistory: Array<{id, traceId, videoTitle, resolution, startedAt}>`.
  - Each entry represents a past playback session.
  - `resolution` is a string like `"RESOLUTION_1080P"` (mapped to "1080p"
    display).
  - `startedAt` is an ISO-8601 timestamp.

## Behaviour

### Query execution

- Component calls `useLazyLoadQuery<TraceHistoryTabQuery>(SESSIONS_QUERY, {},
  { fetchPolicy: "store-and-network" })`.
- Suspends until data arrives (wrapped in page-level `<Suspense>`).
- On success, renders table or empty state.

### Copy button

- Click calls `navigator.clipboard.writeText(traceId)`.
- On success:
  - Button text changes to "Copied".
  - Button applies `styles.copyBtnDone` (green).
  - After 1500ms, resets to "Copy" and neutral styling.
- On failure (e.g., clipboard unavailable): silent failure (no error toast).

### Resolution mapping

Hardcoded `RESOLUTION_LABELS` map:
- `"RESOLUTION_240P"` ‚Üí `"240p"`
- `"RESOLUTION_360P"` ‚Üí `"360p"`
- ‚Ä¶ up to `"RESOLUTION_4K"` ‚Üí `"4K"`.
- Fallback: renders raw resolution string if not in map.

### Time formatting

`formatTime(iso: string)` ‚Äî parses ISO-8601 and formats as:
- Locale-aware (uses `toLocaleString()`).
- Format: `month: "short"`, `day: "numeric"`, `hour: "2-digit"`,
  `minute: "2-digit"`.
- Example: "May 3, 14:30".

## Data

Relay query `TraceHistoryTabQuery` fetches playback sessions from the server's
`playback_history` table (seeded by the streaming service when playback ends).
No mutations. Query is non-cached and always hits the network (via
`store-and-network` policy).

## Notes

### Architectural exception

`TraceHistoryTab` uses `useLazyLoadQuery()` and is wrapped in a page-level
`<Suspense>` boundary instead of colocating its `<Suspense>` inside the tab
component. This is documented in
`docs/code-style/Client-Conventions/00-Patterns.md` as an exception ‚Äî the tab
is expensive (requires network round trip) and should not block the initial
render of lighter tabs (Library, Flags, etc.). The page shell suspends only
when the user navigates to the Trace tab (via URL param).

### Observability integration

- Trace IDs are UUIDs generated by the server (OpenTelemetry trace context).
- Users copy a trace ID and paste it into the Seq UI to inspect all spans and
  events for that session (transcoder start, chunk encoding, buffer underruns,
  etc.).
- See `docs/architecture/Observability/` for full context threading and Seq
  query patterns.

### Strings

Centralized in `TraceHistoryTab.strings.ts` (LocalizedStrings). No i18n
translation strings for resolution or time format (hard-coded).
